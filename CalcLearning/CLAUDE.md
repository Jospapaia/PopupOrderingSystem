# CLAUDE.md — דינו-חשבון 🦕

מדריך לעבודה על הפרויקט. נכתב בעברית כי הפרויקט מתנהל בעברית עם המשתמש ובנו.

## מה זה

משחק לימודי אינטראקטיבי לחיזוק החשבון והגיאומטריה של **יותם** (בן ~9, עולה לכיתה ד').
בנוי כאפליקציית ווב שרצה בדפדפן. הנושא שיותם בחר: **דינוזאורים**. המיקוד ההתחלתי: **כפל וחילוק**.
מכשיר היעד העיקרי: **טאבלט** (מגע, כפתורים גדולים, RTL).

**קהל היעד הוא ילד** — כל שינוי צריך להישאר פשוט, מעודד, וללא תסכול. שפת הממשק: עברית.

## ערימה טכנית

- **Vite + React 18** (JavaScript, לא TypeScript). בלי שרת ובלי מסד נתונים בשלב הנוכחי.
- שמירת התקדמות: `localStorage` (כרגע), מתוכנן לעבור ל-API בעתיד.
- אין קריאות רשת בזמן ריצה. חילול התרגילים **מקומי ודטרמיניסטי** — לא Claude API.

## פקודות

```bash
npm run dev      # שרת פיתוח על 0.0.0.0:5173 (נגיש גם מהטאבלט ברשת המקומית)
npm run build    # בניית production ל-dist/
npm run preview  # תצוגה מקדימה של הבנייה
```

הטאבלט מתחבר דרך כתובת ה-Network שמודפסת ע"י `npm run dev` (למשל `http://10.100.102.7:5173/`).

## מבנה הקוד

| קובץ | אחריות |
|------|--------|
| `src/game.js` | לוגיקה לימודית: מחולל התרגילים, רמות קושי, נתוני הדינוזאורים. **כאן מוסיפים נושאים חדשים.** |
| `src/App.jsx` | רכיבי React: מסכי Welcome / Home / Play + מסך בקיעת דינו. |
| `src/storage.js` | שכבת שמירת ההתקדמות. **נקודת החיבור היחידה למעבר לשרת.** |
| `src/styles.css` | עיצוב מותאם-טאבלט (כפתורים גדולים, RTL, אנימציות). |

## החלטות עיצוב (ולמה)

- **חילול תרגילים מקומי, לא Claude API** — מיידי, חינם, עובד אופליין, בלי מפתח חשוף באפליקציית ילדים.
  חילוק תמיד מיוצר כפעולה הפוכה של כפל כדי שהתוצאה שלמה (בלי שארית).
- **`storage.js` מבודד** — מבנה ה-`profile` תוכנן להתמפות 1:1 לשרת. מעבר למרובה-משתתפים = החלפת
  `loadProfile`/`saveProfile` בקריאות `fetch` בלבד, בלי לגעת בשאר המשחק.
- **מוטיבציה מעל תסכול** — 4 תשובות לבחירה (לא הקלדה), משוב מעודד, רצף, מטבעות, ובקיעת דינוזאור
  כרגע-שיא כל 5 תשובות נכונות (`CORRECT_PER_LEVEL`).
- **קושי מדורג** — לוחות 2,5,10 בהתחלה ומתרחב עד 9. ראה `tablesForLevel` ב-`game.js`.

## מפת דרכים

1. **עכשיו:** לשחק, לאסוף משוב מיותם, לשפר באיטרציות.
2. **שלב 2 — שיתוף (על שרת Hetzner קיים של המשתמש):** שרת קטן (Node/Express או FastAPI) עם
   פרופילי שחקנים, לוח תוצאות בין חברים, ואולי אתגר יומי משותף. מחליפים רק את `storage.js`.
3. **הרחבת תוכן:** גיאומטריה (צורות, זוויות, היקף/שטח, סימטריה), שברים, בעיות מילוליות.
   בבעיות מילוליות אפשר לשקול תוכן מיוצר-מראש (אולי בעזרת Claude API, offline) — לא קריאות חיות.

## הקשר משתמש

- המשתמש = אבא של יותם, ראש צוות פיתוח תוכנה. חלק מהמטרה: להראות ליותם מה אבא עושה בעבודה,
  אז שווה להסביר תוך כדי מושגים כמו איטרציות, MVP, ומבנה קוד — בשפה פשוטה.
- כשיותם נוכח, לענות בעברית.
